Standard-Curve Pool Mode

Per pool (default): a curve is fit and shown for every control pool × antigen with no antigen→pool matching; specimen RAU / range use the single scoring pool above. Auto-select: each antigen is matched to its pathogen's pool (tie-broken by best fit) and scored against that one.

Pool assignment rules (auto-select mode; optional)

The regex is matched against the antigen name; the pool name must match one of the plate's control pools (shown in the report's Standard-Curve Summary). Used only in Auto-select mode. Lets you define the antigen→pool mapping explicitly instead of relying on the built-in keyword heuristic.

How the regex works (quick guide)
  • The pattern is matched anywhere within the antigen name, and is case-sensitive (e.g. DENV matches ARB_DENV2_VLP).
  • . = any single character; .* = any run of characters (including none). So ARB_DENV.* matches every antigen whose name contains ARB_DENV.
  • | = OR. CHO_.*|.*CtxB.* matches names with CHO_ or CtxB.
  • ^ = start of the name, $ = end. Use ^CHO_ to match only names that begin with CHO_.
  • Escape special characters with a backslash — e.g. S\.typhi to match a literal dot in BAC_S.typhi_HlyE.
  • Rules are tried top to bottom; the first match wins. Put the most specific rules first.
Examples:
ARB_DENV.*  => Dengue pool
^CHO_.*|.*CtxB.* => Anti-OSP & cTxB pool
.*HlyE.*  => HlyE
How antigens are matched to pools (full detail)

This matching applies only in Auto-select mode. For each antigen the tool decides which control pool's curve to score it against by working down this list and stopping at the first step that yields a pool:

  1. Exact override — a per-antigen entry in panel.pool_antigen_overrides in config.yaml (advanced; not editable above). Wins outright, provided that pool produced a fit for the antigen.
  2. Your regex rules (the box above) — the first rule whose pattern matches the antigen name, provided that pool fit the antigen.
  3. Built-in pathogen keyword match — the tool reads the pathogen from the antigen's name and keeps the pools that target it (table below) and that actually fit the antigen.
  4. Best-fit fallback — if nothing above matches, any pool that produced a usable fit for that antigen.

Built-in keyword map (used in step 3):

  • Dengue — antigen name contains DENV or DENGUE → pools whose name contains dengue or orpal.
  • Cholera — antigen starts with CHO_ or contains CtxB / Inaba / Ogawa / cholera / vibrio → pools whose name contains osp, ctxb, or cholera.
  • Typhoid — antigen contains HlyE or typhi → pools whose name contains hlye.

When more than one pool matches (e.g. both Dengue pool and Orpal pool target dengue), exactly one is still used — the best-fitting curve, ranked by: a curve was fit → it passed fit QC (fit_ok) → higher R². An antigen with no usable fit in any pool is left NO_FIT. The chosen pool appears in the report's Standard-Curve Summary Pool column, so you can confirm the result and add a regex rule (or override) where the keyword guess is wrong.

In Per pool mode none of this runs — every antigen is scored against the single scoring pool above, and the master export lists RAU under every pool so you can choose per antigen downstream.

Well Classification

Wells are classified from the Sample name in the CSV (matched with these regexes, in order: Background → NC → PC → specimen). Defaults fit the Bangladesh naming (Background0, Pilot Control: …, Pilot Control: Negative …, {id}_r3_{Serum|DBS}).

NC is checked before PC because NC and PC samples share the Pilot Control: prefix, so the more specific Negative match wins. Anything matching none of the patterns is treated as a specimen.

Standard Curve

Dilutions are parsed automatically from each control sample name — 1:N for the pooled series (e.g. Pilot Control: Dengue pool 1:4000) and N ng/mL for HlyE. A 4PL is fit per (control pool × antigen) using that pool's own series, so there is no fixed dilution list to configure here. Single-point controls (e.g. Cholera High/Low) are shown as reference markers, not fit.

QC Thresholds

Bead count tiers (RED/YELLOW/GREEN) and the problem-fraction threshold drive the Bead Count cards: an antigen (resp. specimen) is flagged when at least this fraction of its wells (resp. antigens) is red or yellow. The same fraction is applied to BELOW_RANGE / ABOVE_RANGE counts in the Standard-Curve Summary. Recovery tolerance sets the standard-curve reportable range (LLOQ/ULOQ). The two Background values are tracked for reference only — formal Background pass/fail flagging is still in development, so they do not currently mark antigens as failing.

If a standard curve fails QC, try removing each point one at a time (leave-one-out). If dropping a single point produces a passing fit, use that fit and flag the dropped point. Requires at least 6 remaining points.
